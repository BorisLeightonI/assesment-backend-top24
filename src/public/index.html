Documentación

Las propiedades necesarias para el registro de usuarios como de listas FavList y de Favs queda sujeto a las restricciones señaladas en el documento original.

Rutas base

End-point Colección
/api/favs/ fav
/api/favLists/ favList
/auth/local/ user

Modo de uso

1. user

Registro de nuevo usuario, signup: Petición POST a ruta /auth/local/signup

Login de usuario, signin: Petición POST a ruta /auth/local/signin

2. fav

Listar todos los fav: Petición GET a ruta /api/favs

Listar un fav: Petición GET a ruta /api/favs/:favId donde favId representa el ID que retorna mongo atlas

Crear un fav: Petición POST a ruta /api/favs/:favListId donde favListId representa el ID que retorna mongo atlas de la Lista de Favs que desea vincular

Modificar un fav: Petición PUT a ruta /api/favs/:favId donde favId representa el ID que retorna mongo atlas del respectivo Fav

Eliminar un fav: Petición DELETE a ruta /api/favs/:favId donde favId representa el ID que retorna mongo atlas del respectivo Fav

3. favList

Listar todos las favList: Petición GET a ruta /api/favLists

Listar un favList: Petición GET a ruta /api/favLists/:favListId donde favListId representa el ID que retorna mongo atlas

Crear un favList: Petición POST a ruta /api/favLists/:userId donde userId representa el ID que retorna mongo atlas de usuario que desea vincular

Modificar un favList: Petición PUT a ruta /api/favLists/:favListId donde favListId representa el ID que retorna mongo atlas del respectivo FavList

Eliminar un favList: Petición DELETE a ruta /api/favLists/:favListId donde favListId representa el ID que retorna mongo atlas del respectivo FavList
